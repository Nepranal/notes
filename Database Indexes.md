# Clustered, Non-Clustered, & Composite Index

Indexes give structures to stored data in a DB. There are 3 types of interest: clustered, non-clustered, and composite index. I should mention that these types of indexes can typically be formed in the same manner. A clustered index is when the data being stored is structured in a particular way. A non-clustered index is to have a reference to those data being structured in a particular way. Finally, the composite index uses multiple columns of the database table to create an indexing.

The 2 main types here are the clustered related one. Because clustered indexes structure the data itself, you can only use one of them in comparison with non-clustered indices since they can just create new references.
